

# План: Видимая кнопка удаления на карточках обсуждений

## Текущее состояние

Сейчас кнопка удаления скрыта в dropdown-меню, которое появляется только при hover. На мобильных устройствах это неудобно.

## Решение

Заменить скрытый dropdown на **всегда видимую кнопку удаления** справа на карточке.

---

## Визуальный результат

```text
До (hover required):
┌────────────────────────────────────────┐
│ Новое обсуждение                    →  │  (кнопка ⋮ появляется при hover)
│ 11 минут назад • Нет источников        │
└────────────────────────────────────────┘

После:
┌────────────────────────────────────────┐
│ Новое обсуждение              [🗑] →   │  (кнопка всегда видна)
│ 11 минут назад • Нет источников        │
└────────────────────────────────────────┘
```

---

## Изменения

### Файл: `src/components/discussions/SessionCard.tsx`

1. **Удалить** весь блок DropdownMenu (строки 71-96)
2. **Добавить** видимую кнопку удаления рядом с ChevronRight:

```tsx
{/* Delete button - always visible */}
<Button
  variant="ghost"
  size="icon"
  className="h-7 w-7 shrink-0 text-muted-foreground/50 hover:text-destructive hover:bg-destructive/10"
  onClick={(e) => {
    e.preventDefault();
    e.stopPropagation();
    onDelete?.(session.id!);
  }}
>
  <Trash2 className="h-4 w-4" />
</Button>

<ChevronRight className="..." />
```

3. **Изменить** структуру Link — добавить кнопку в flex-контейнер

---

## Упрощения

- Убираем DropdownMenu — меньше кода
- Убираем "Pin/Unpin" из интерфейса (редко используется)
- Подтверждение удаления остается в AlertDialog на странице списка
